Visão geral

Padrões de campos de formulário alinhados a componentes.css: .form-group, .control-label e .form-control em input, select e textarea (incluindo o bloco Textarea · comentários), além de interruptores no painel de filtros (secção Switch: isolado, grupo com select). Em muitas views MVC o Bootstrap ainda fornece reset adicional; a base em componentes.css cobre o aspecto Invoisys (altura ~39px, borda, sombra interior).

Prioridade em listagens e filtros: trate Select2 e intervalo de datas (Moment.js + bootstrap-daterangepicker, campo «Data de exibição» nas grades) como integrações do mesmo patamar — ambas dependem de jQuery, CSS vendor, ordem correta de scripts e do stack Invoisys (componentes.css, table.css). O intervalo segue o mesmo arranjo visual que o select avançado (rótulo, botão de ajuda, borda verde marca ao abrir o painel).

Select2: o aspecto não vem só do select2.min.css — há CSS exclusivo Invoisys que completa o componente (detalhe na secção Select2 e no snippet de init).

Datas: uma data única pode usar input type="date" (Data nativa). Para intervalos com presets e calendário duplo, siga a secção Intervalo de datas (prioridade acima).

Texto

type="text", email, url, etc. — mesma classe form-control. Para pesquisa na toolbar ou listagens, ver Pesquisa (search).

Textarea · comentários

textarea.form-control com várias linhas — em componentes.css o textarea partilha borda e foco com os input, com altura mínima própria. O padrão abaixo replica o bloco habitual na ficha (ex.: NF-e): cabeçalho com ícone Font Awesome + rótulo, campo com placeholder e ação primária alinhada à direita (btn btn-primary — ver página Botões).

HTML / Razor (textarea + botão) 
<div class="form-group">
    <label class="control-label d-flex align-items-center gap-2 mb-2" for="ComentarioTexto">
        <i class="fa fa-comment-o" aria-hidden="true"></i>
        <span>Comentários</span>
    </label>
    <textarea class="form-control" id="ComentarioTexto" name="ComentarioTexto" rows="5" placeholder="Adicionar um comentário…"></textarea>
    <div class="d-flex justify-content-end mt-2">
        <button type="submit" class="btn btn-primary m-0">Adicionar</button>
    </div>
</div>

Número

type="number" — útil para quantidades; em Razor você pode usar Html.TextBoxFor com tipo ou atributos.

Switch (painel de filtros)

O interruptor SIM / NÃO das colunas de filtro usa a marcação .switch com .switch-input (checkbox), .switch-label e .switch-handle. Em componentes.css, os ajustes de tamanho e texto aplicam-se dentro de #filtersList (painel de filtros ao lado da grade). O aspecto completo do trilho e do texto costuma completar-se com o CSS global do portal (ex.: ui.css no bundle MVC).

Storybook: na mesma view MVC mantém um único id="filtersList" — não dupliques. Aqui há dois exemplos em caixas separadas: o primeiro usa filtersList (como em produção); o segundo usa id="filtersListGrupo" só para esta página não ter HTML inválido — ao copiar o snippet do grupo, troca para filtersList se for o mesmo painel que o caso isolado, ou integra o markup dentro do teu #filtersList real.

Uma linha: texto à esquerda, interruptor à direita — típico para um único critério binário (ex.: incluir filiais) sem outro controlo no mesmo form-group.

Data nativa

input type="date" — calendário nativo do browser, sem Moment nem plugin. Útil para uma única data quando não precisa de presets nem de intervalo no mesmo controle.

Pesquisa (search)

input type="search" ou classe .search-input — campo de pesquisa ao lado de filtros e grades. Em table.css as regras .search-input e input[type="search"] alinham altura (~39px) e tipografia ao resto da toolbar (junto de .date-range-input).

Na view MVC com listagens: inclua table.css na cascade (ver Introdução — ordem de cascade nas views). O storybook também carrega table.css; o exemplo abaixo usa ainda design-system-docs.css (.invoisys-ds-search-demo) para isolar padding e sombra do demo sem depender do contexto completo da toolbar.

Arquivo (file)

No portal, o padrão mais comum para XML ou anexos é o campo composto: caixa única com fundo #fafafa, texto só leitura com o nome (ou «Nenhum arquivo selecionado») e botão verde Escolha o … à direita. Use .file > .option-group com input.form-control (readonly), span.file-button.btn-primary e input type="file" com classes .dropzone.custom-file — estilos em componentes.css (o input file fica invisível por cima do botão para o clique abrir o diálogo).

Boas práticas: accept para extensões/MIME esperados, aria-label no type="file" e no campo de nome, onchange (ou JS) para copiar o nome do arquivo para o readonly, e validação no servidor sempre. Ver também o modal Entrada XML na página Modais.

Escolha o XML
HTML / Razor (referência) 
<div class="form-group">
    <label for="arquivoXMLPost" class="control-label">Arquivo XML</label>
    <div class="file">
        <div class="option-group">
            <input type="text" class="form-control" id="uploaderXML" placeholder="Nenhum arquivo selecionado" readonly="readonly" aria-label="Nome do arquivo selecionado" />
            <span id="btnXML" class="file-button btn-primary">Escolha o XML</span>
            <input id="arquivoXMLPost" name="arquivoXMLPost" type="file" class="dropzone custom-file" accept=".xml,application/xml,text/xml"
                   onchange="document.getElementById('uploaderXML').value = this.files && this.files[0] ? this.files[0].name : '';"
                   aria-label="Selecionar arquivo XML" />
        </div>
    </div>
</div>

Variação mínima: input type="file" só com .form-control — o botão «Procurar…» é o nativo do browser (aspecto inconsistente entre sistemas).

Select (nativo)

select.form-control — opção única, sem JavaScript: o menu que abre é o do browser (no macOS costuma mostrar o visto ✓ nas opções). Isto não é Select2. Variantes como .select-filter / .select-verde estão em componentes.css para estados específicos.

Multiselect (nativo)

select multiple com size — útil como fallback sem JavaScript. Para pesquisa, chips e múltipla seleção alinhada ao resto do portal, use Select2 na seção seguinte.

Select2 (biblioteca + CSS Invoisys) Prioridade

O Select2 é uma biblioteca de terceiros: substitua o <select> por markup próprio (.select2-container, .select2-selection, …). O portal carrega o CSS base do pacote e, a seguir, as folhas Invoisys que personalizam esse componente — não basta o arquivo da biblioteca sozinho.

Ordem típica no _Layout.cshtml / bundles (MVC):

  1. CSS do Select2 — arquivo distribuído com a biblioteca (ex.: select2.min.css em Content/Css/vendor/ ou via CDN). Em bundles antigos o import costumava chamar-se select2_dist_css_select2.min.css — é o mesmo papel: folha base do pacote antes dos overrides Invoisys.
  2. Design system — main.css, tokens.css, componentes.css (regras globais .select2-container, .select2-selection--single, .select2-selection--multiple, empresas, filtros, etc.).
  3. Listagens / DataTables — table.css quando o select está no contexto de grade (ex.: toolbar com loading).
  4. CSS por tela — quando existir, folha em Content/Css/Pages/… com seletores específicos (ex.: Delivery Monitor, filtros com classes dedicadas no dropdown).

JavaScript: jQuery + select2.min.js (ou bundle); na view, $(\'#MeuSelect\').select2({ … }) — o mesmo padrão dos scripts em Scripts/ do projeto.

O primeiro exemplo reproduz o aspecto típico do portal: rótulo + ajuda contextual, placeholder longo, campo de pesquisa na lista aberta e borda verde marca unindo a caixa ao painel de resultados (detalhe tratado no CSS da solução + overrides em componentes.css / página).

Layout e init (snippet)

Links + init mínimo · prioridade (listagens) 
<!-- _Layout.cshtml — ordem sugerida (antes de componentes.css) -->
<link href="~/Content/Css/vendor/select2.min.css" rel="stylesheet" />
<!-- … main.css, tokens.css, componentes.css, table.css se aplicável, CSS da página … -->

<!-- Scripts (jQuery antes de select2) -->
<script src="~/Scripts/vendor/jquery.min.js"></script>
<script src="~/Scripts/vendor/select2.min.js"></script>

<!-- View: o <select> mantém classes habituais; Select2 injeta o resto -->
<select id="CidadeId" name="CidadeId" class="form-control">…</select>
<script>
  $(function () {
    $('#CidadeId').select2({ width: '100%', placeholder: 'Selecione…' });
  });
</script>

Intervalo de datas (biblioteca + CSS Invoisys) Prioridade

O bootstrap-daterangepicker é biblioteca de terceiros (jQuery + Moment.js): o filtro «Data de exibição» nas invoices e listagens é um input type="text", não type="date". Trate-o como integração crítica ao lado do Select2 (mesma cascade de vendors + main.css / tokens.css / componentes.css / table.css). O portal carrega o CSS do pacote e, em seguida, o mesmo stack Invoisys que o Select2 (table.css para .date-range-input / .date-range-container, componentes.css para .prepend-icon) — o aspecto do campo na toolbar fica coerente com os selects Select2 (altura ~39px, borda neutra, borda verde marca ao focar/abrir).

Ordem típica no _Layout.cshtml / bundles (MVC) — espelha a seção Select2 e adicione o vendor do intervalo:

  1. CSS do daterangepicker — daterangepicker.css (ou min) em Content/Css/vendor/. Na mesma página com Select2, mantenha primeiro o select2.min.css (passo 1 da lista Select2) e coloca este arquivo a seguir, ainda antes de main.css.
  2. Design system — main.css, tokens.css, componentes.css (inclua .prepend-icon + restantes inputs) e table.css para regras .date-range-* nas toolbars.
  3. Listagens / DataTables — reforço de contexto em table.css quando o intervalo está na grade (igual ao Select2 na toolbar).
  4. CSS por tela — Content/Css/Pages/… quando a tela ajusta o bloco (ex.: Delivery Monitor). Combinações como «Classificar por data de: Entrada» são HTML extra da tela, não fazem parte do plugin.

JavaScript: jQuery → Moment (moment.min.js) → daterangepicker (daterangepicker.min.js); na view, $(\'#MeuIntervalo\').daterangepicker({ … }) — o mesmo padrão de ordem que jQuery antes de select2.min.js na seção Select2.

O exemplo abaixo segue o mesmo arranjo visual da demo Select2 (rótulo + ajuda + campo) para filtros de listagem; o painel do calendário usa a verde marca como o dropdown Select2 nesta documentação.

Layout e init (snippet)

Links + init mínimo · prioridade (listagens) 
<!-- _Layout.cshtml — vendors (Select2 + intervalo) antes de main/tokens/componentes/table -->
<link href="~/Content/Css/vendor/select2.min.css" rel="stylesheet" />
<link href="~/Content/Css/vendor/daterangepicker.css" rel="stylesheet" />
<!-- … main.css, tokens.css, componentes.css, table.css, CSS da página … -->

<script src="~/Scripts/vendor/jquery.min.js"></script>
<script src="~/Scripts/vendor/moment.min.js"></script>
<script src="~/Scripts/vendor/daterangepicker.min.js"></script>
<script src="~/Scripts/vendor/select2.min.js"></script>

<!-- View: markup alinhado a table.css + componentes.css (prepend-icon) -->
<div class="date-range-container">
  <div class="prepend-icon dm-toolbar-date-range">
    <i class="fa fa-calendar"></i>
    <input type="text" class="form-control date-range-input" id="dmDateRange" autocomplete="off" />
  </div>
</div>

<script>
  $(function () {
    if (!$('#dmDateRange').length || typeof $.fn.daterangepicker !== 'function' || typeof moment === 'undefined') return;

    $('#dmDateRange').daterangepicker({
      showDropdowns: false,
      autoApply: true,
      linkedCalendars: true,
      autoUpdateInput: false,
      startDate: moment(),
      endDate: moment(),
      ranges: {
        'Ontem': [moment().subtract(1, 'days'), moment().subtract(1, 'days')],
        'Hoje': [moment(), moment()],
        'Semana Atual': [moment().startOf('week'), moment().endOf('week')],
        'Mês Atual': [moment().startOf('month'), moment().endOf('month')],
        'Semana Passada': [moment().subtract(1, 'week').startOf('week'), moment().subtract(1, 'week').endOf('week')],
        'Mês Passado': [moment().subtract(1, 'month').startOf('month'), moment().subtract(1, 'month').endOf('month')]
      },
      locale: {
        format: 'DD/MM/YYYY',
        separator: ' - ',
        applyLabel: 'Aplicar',
        cancelLabel: 'Limpar',
        fromLabel: 'De: ',
        toLabel: 'Até: ',
        daysOfWeek: ['Dom', 'Seg', 'Ter', 'Qua', 'Qui', 'Sex', 'Sáb'],
        monthNames: ['Janeiro', 'Fevereiro', 'Março', 'Abril', 'Maio', 'Junho', 'Julho', 'Agosto', 'Setembro', 'Outubro', 'Novembro', 'Dezembro'],
        firstDay: 0
      },
      alwaysShowCalendars: true
    });

    var hoje = moment();
    $('#dmDateRange').val(hoje.format('DD/MM/YYYY') + ' - ' + hoje.format('DD/MM/YYYY'));

    $('#dmDateRange').on('apply.daterangepicker', function (e, picker) {
      $(this).val(
        picker.startDate.format('DD/MM/YYYY') + ' - ' + picker.endDate.format('DD/MM/YYYY')
      );
    });
  });
</script>
