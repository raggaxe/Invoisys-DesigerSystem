Tabelas · utilitários

Opções que costumam ir no segundo argumento de startTable (e o terceiro, quando há busca), além de trechos de UI na toolbar. O comportamento canônico está em static/scripts/Tabelas/table-setting.js; exemplos reais aparecem nos scripts de cada tela (ex.: static/scripts/table-init.js no Delivery Monitor).

Antes de usar isto, confira a Tabelas · visão geral (data-table, ordem de scripts e contrato do painel) e, se estiver montando a toolbar, Tabelas · cabeçalho da listagem.

serverSide e ajax

O motor define serverSide: true e processing: true por padrão. Na tela, passe ajax com url, type (ex.: 'POST') e função data para juntar filtros ao payload que o DataTables envia ao servidor.

  • Grade só no cliente — defina serverSide: false e use data (array/objeto) em vez de ajax, se fizer sentido para a tela.
  • Recarregar dados — com API do DataTables: $('#idDaTabela').DataTable().ajax.reload(null, false); (o segundo argumento false mantém a página atual; use true para voltar à primeira página).
Ajax + colunas (trecho ilustrativo) 
var options = {
  serverSide: true,
  ajax: {
    url: '/MinhaArea/Lista/DataTable',
    type: 'POST',
    data: function (d) {
      d.status = $('#filtroStatus').val();
      return d;
    }
  },
  columns: [
    { data: 'id', name: 'id', title: 'ID' },
    { data: 'nome', name: 'nome', title: 'Nome' }
  ]
};
startTable('#tblMinhaLista', options);

Campo de busca (search custom)

O terceiro argumento de startTable(lookup, options, searchSelector) é um jQuery do input de pesquisa. Se omitir, o motor tenta $('#customSearch').

Por padrão o DataTables fica com searching: false no merge global; o texto digitado chama dataTable.search(valor).draw(), o que com serverSide: true dispara nova requisição a cada alteração (comportamento típico de “busca no servidor”).

HTML + startTable com seletor explícito 
<input type="search" id="minhaBuscaGrade" class="form-control" placeholder="Pesquisar…" />

<script>
  startTable('#tblMinhaLista', options, $('#minhaBuscaGrade'));
</script>

searchClientSideOnly — filtro só na página atual

Com searchClientSideOnly: true nas opções (não é opção nativa do DataTables; o motor remove antes do $.extend final), a busca não chama search().draw() no servidor. Em vez disso, filtra apenas as linhas já desenhadas na página atual (útil quando a pesquisa principal é outra ou os dados já estão na viewport).

Exemplo 
startTable('#tblMinhaLista', {
  serverSide: true,
  searchClientSideOnly: true,
  ajax: { /* … */ },
  columns: [ /* … */ ]
}, $('#filtroRapidoLocal'));

Paginação dinâmica (invTableUi)

O número de linhas por página inicial é calculado pela altura útil da área da tabela (getDynamicPageLength). Ajuste isso com o objeto invTableUi dentro de options (o motor funde com defaults e remove antes de passar ao DataTables).

  • pageLengthMenu — lista de valores do menu “resultados por página” (ex.: [20, 50, 100, -1]; -1 = “Todos”).
  • pageLengthClamp — { min: 10, subtractAfterMeasure: 1 } combina mínimo com o valor medido (útil quando sobra uma linha “cortada”).
  • dynamicPageLengthMode — 'measureFooter' (mede .footer-table) ou 'fixedReserve' (reservas fixas para layouts mais densos).
  • Override direto — pode passar pageLength e/ou lengthMenu no mesmo nível das opções do DataTables; o motor respeita quando a chave vem explicitamente do script da tela.
invTableUi ilustrativo 
startTable('#tblMinhaLista', {
  ajax: { /* … */ },
  columns: [ /* … */ ],
  invTableUi: {
    dynamicPageLengthMode: 'measureFooter',
    pageLengthMenu: [15, 30, 60, 100, -1],
    pageLengthClamp: { min: 10, subtractAfterMeasure: 1 }
  }
});

ColVis — “Editar colunas”

O motor já inclui um botão Buttons colvis (“Editar colunas”) e colunas com classe .noVis podem ser excluídas da lista via columnDefs padrão (columns: ':not(.noVis)').

Para abrir o ColVis a partir de um botão quadrado na toolbar (ícone de colunas / elipse), passe colvisTriggerSelector com o seletor do contêiner (ex.: '#btnOcultarColunas'). O motor injeta 'B' no dom, move .dt-buttons para dentro do trigger e usa a classe colvis-trigger-container no markup — alinhado ao padrão das telas com .square-button em table.css / toolbar.

Opções + markup mínimo do trigger 
<div id="btnOcultarColunas">
  <button type="button" class="square-button" aria-label="Editar colunas visíveis">
    <i class="fa fa-bars" aria-hidden="true"></i>
  </button>
</div>

<script>
  startTable('#tblMinhaLista', {
    colvisTriggerSelector: '#btnOcultarColunas',
    ajax: { /* … */ },
    columns: [ /* … */ ]
  });
</script>

Algumas telas substituem o dropdown nativo por painel próprio (checkboxes, etc.) — nesse caso mantenha a lógica na tela e alinhe com o time antes de duplicar padrões.

Botão “Atualizar” (ícone de refresh)

Use um button.square-button (ou o padrão de toolbar da tela) com ícone fa fa-refresh e, no clique, chame ajax.reload na instância do DataTables.

Markup + handler 
<button type="button" id="btnAtualizarGrade" class="square-button" aria-label="Atualizar listagem">
  <i class="fa fa-refresh" aria-hidden="true"></i>
</button>

<script>
  $('#btnAtualizarGrade').on('click', function () {
    var api = $('#tblMinhaLista').DataTable();
    if (api && api.ajax) api.ajax.reload(null, false);
  });
</script>

Estilos do botão quadrado seguem o design system — ver também a página Botões para hierarquia com .panel-card / toolbar.

Seleção de linhas (click to select)

O merge padrão usa extensão Select com style: 'multi', items: 'row', classe de linha selecionada bg-row-selected e seletor td.selectable — ou seja, o clique na linha só conta onde houver célula com essa classe.

  • Desligar seleção — select: false nas opções da tela.
  • Checkbox mestre — o motor sincroniza ícones #selectRows / #checkUncheck com a API quando existem no painel; funções auxiliares como setupMasterCheckbox estão no mesmo arquivo para reuso.
  • Manter seleção em redraws — invTableUi.keepSelectionOnRedraw: true quando fizer sentido (cenários com overlay e draw frequentes).

Barra de ações e buttons

A disposição em duas linhas (filtro data, pesquisa, KPIs, ações, segunda linha com “Filtrar por”) está em Tabelas · cabeçalho da listagem. A toolbar combina .panel-card, .square-button e áreas de filtro documentadas em Inputs e Cabeçalho (componente de app, não confundir com este painel de tabelas). No DataTables, o array buttons do motor já traz ColVis; só use buttons: [] se a tela substituir o ColVis do plugin por UI própria (senão mantenha o default ou estenda o array).

Outros botões do plugin (exportar, etc.) seguem a mesma mecânica do Buttons — acrescente entradas no array e ajuste o dom apenas se o time padronizar outro layout (o dom padrão do motor já reserva a área de scroll + rodapé).

Outras opções úteis

  • ordering — controle de ordenação por coluna; desative globalmente ou por coluna conforme a API do DataTables.
  • invTableUi.deferInitialTableReveal — atrasa mostrar .table-wrap até o primeiro desenho (loading visível mais tempo no arranque).
  • invTableUi.afterDraw / onDataTableError — ganchos opcionais para DOM pós-draw ou tratamento de erro de rede/API.
  • invTableUi.processingOverlaySuppressFlag — nome de variável global: quando true durante processing, o motor pode suprimir o overlay de loading (padrão usado em fluxos muito específicos).
