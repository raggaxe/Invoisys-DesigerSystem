Tabelas · visão geral

Listagens com DataTables, estilos em table.css e um motor JS em duas camadas: configuração global compartilhada por todas as grades (table-setting.js) e inicialização por tela, onde cada tela define colunas, fonte de dados e comportamentos próprios (ex.: table-init.js ou um arquivo por funcionalidade).

No projeto MVC, alinhe pastas ao Visual Studio: Content/Css para table.css e restantes folhas do design system; Scripts/Tabelas (ou a pasta que o projeto usar) para o motor comum e para os scripts de cada listagem. Para a toolbar em duas linhas (datas, pesquisa, KPIs, ações, segundo filtro, “Filtrar por”), veja Tabelas · cabeçalho da listagem. Para cabeçalho e grade no mesmo exemplo, veja Tabelas · listagem completa. Para busca custom, ajax, paginação dinâmica, ColVis, botão de atualizar, serverSide e seleção de linhas, veja Tabelas · utilitários.

Atenção: inicialização do data-table

Coloque data-table="#idDaTabela" no section.panel ou div.panel que envolve toda a listagem (toolbar, filtros, área da tabela e rodapé) — não só na <table>. Sem isso, o motor não resolve o contêiner e o table.css (altura, rolagem, rodapé) deixa de casar com o HTML.

Regra: o valor de data-table e o primeiro argumento de startTable('#…', …) precisam ser o mesmo #id da tabela.

Documentação relacionada: Tutorial nesta página, Introdução e Quick start.

Tutorial · inicializar a grade

Em qualquer listagem do portal o fluxo é o mesmo: HTML com o painel certo → table-setting.js (global) → script da tela que chama startTable com colunas e ajax.

  1. Marcação — use layout-datatable-page na página, um painel com data-table="#…" envolvendo toolbar (se houver), área da tabela e rodapé da grade. Abaixo há um exemplo estático (10 linhas fictícias) só para copiar a estrutura.
  2. JavaScript — na view, carregue table-setting.js depois do jQuery/DataTables e, em seguida, o JS da tela (ex.: table-init.js ou um arquivo por funcionalidade). Quando o DOM estiver pronto, chame startTable('#mesmoId', opções [, campoDeBusca]).
  3. CSS — inclua table.css na ordem certa da view; detalhes na secção CSS de tabela na view.

Motor global — table-setting.js

Este arquivo concentra o que é igual em todas as tabelas: textos de idioma, dom do DataTables (área de scroll, rodapé com info, paginação e “resultados por página”), estados vazios (sem registros / pesquisa sem resultado), loading sobre a .table-wrap, integração com seleção de linhas e checkbox mestre, botão base de ColVis (“Editar colunas”), e a função startTable que monta a instância final.

  • startTable(lookupSelector, specificOptions [, searchSelector]) — o primeiro argumento (ex.: '#tblMinhaLista') precisa ser o mesmo seletor que em data-table="#tblMinhaLista" no painel (section.panel ou div.panel). O motor resolve [data-table="…"], localiza table.table e faz $.extend das opções globais com as da tela.
  • Padrões de DataTables — por padrão entram opções como serverSide: true, processing: true, searching: false (a pesquisa costuma ser tratada à parte), select multi-linha, lengthMenu com página dinâmica calculada pela altura útil, e textos em português.
  • invTableUi — objeto opcional dentro de specificOptions com chaves só de UI do portal (ex.: modo de cálculo de linhas por página, classes extra no loading, suprimir overlay em certos redraws). O motor remove-o antes de passar o objeto ao DataTables.
  • Exceções por produto — algumas telas recebem defaults extras de invTableUi quando o HTML segue um padrão conhecido; outras combinam atributos no markup com ramos dedicados no mesmo arquivo.
Ordem de scripts na view (Razor) 
<!-- Depois de jQuery e do bundle DataTables / extensões necessárias -->
<script src="~/Scripts/Tabelas/table-setting.js"></script>
<script src="~/Scripts/…/table-init.js"></script><!-- ou o JS específico desta tela -->

Por tela — table-init.js (ou equivalente)

Cada listagem deve ter um script (ou módulo) que só define o que muda: columns, ajax (URL, verbo, função data para filtros e parâmetros do servidor), se a grade é serverSide ou não, pageLength / lengthMenu próprios, ordering, select: false quando não houver seleção, buttons vazio se o ColVis for customizado fora do plugin, initComplete para testes ou ajustes de DOM, e opções avançadas como colvisTriggerSelector ou searchClientSideOnly quando a pesquisa for só nas linhas já desenhadas.

O script da tela chama startTable('#idDaTabela', options [, jQuery('#campoBusca')]) quando o DOM está pronto. O seletor precisa ser o mesmo data-table do painel que envolve a listagem inteira.

Esqueleto ilustrativo (não é uma tela real) 
$(function () {
  if (typeof startTable !== 'function') return;

  var tableLookup = '#tblMinhaLista';

  var options = {
    serverSide: true,
    ajax: {
      url: '/api/minha-lista/datatable',
      type: 'POST',
      data: function (d) {
        d.filtroStatus = $('#filtroStatus').val();
        return d;
      }
    },
    columns: [
      { data: 'id', title: 'ID' },
      { data: 'nome', title: 'Nome' }
    ],
    // invTableUi: { pageLengthClamp: { min: 10, subtractAfterMeasure: 1 }, … }
  };

  startTable(tableLookup, options, $('#minhaBusca'));
});

Referência neste repositório: o motor global está em static/scripts/Tabelas/table-setting.js; um exemplo completo de tela (colunas, POST, paginação dinâmica, pesquisa local, ColVis custom) está em static/scripts/table-init.js (Delivery Monitor). Copie o padrão, não o tamanho do arquivo — para uma nova tela, crie um JS dedicado e mantenha-o o mais enxuto possível.

CSS de tabela na view

Garanta que table.css entra depois de tokens.css e antes ou depois de componentes.css, conforme a cascata do projeto — o importante é não saltar main.css nem duplicar variáveis.

Razor / HTML 
<link href="~/Content/Css/main.css" rel="stylesheet" />
<link href="~/Content/Css/tokens.css" rel="stylesheet" />
<link href="~/Content/Css/table.css" rel="stylesheet" />
<link href="~/Content/Css/componentes.css" rel="stylesheet" />

Head completo numa listagem (DataTables + ícones + Lato): para o aspecto ficar alinhado ao que esta documentação carrega no <head> (e ao portal), acrescente ainda Font Awesome 4.7 (coluna Status, ajuda, etc.), a folha base do DataTables (grade, info e blocos de paginação) e a fonte Lato (mesmos pesos que em Tipografia). O Bootstrap 3 do _Layout global já cobre .breadcrumb, .pagination, .form-inline usados com dt-bootstrap — não carregue outra major na view. Se a toolbar tiver filtros Select2 ou bootstrap-daterangepicker, inclua também essas folhas vendor antes do stack Invoisys (ver Inputs).

<th> e classe th-title-abbrv: o estilo dos cabeçalhos (fundo #F1F1F1, maiúsculas, altura 36px, tipografia, etc.) já está em table.css em .table > thead > tr > th — não precisa repetir esse bloco num <style> na view se table.css estiver carregado (evita duplicação e conflitos). A regra só para reticências table.th-title-abbrv > thead > tr > th { overflow; text-overflow: ellipsis; … } não está em table.css deste repositório; no portal, inclua em um CSS compartilhado ou integre no canônico. No snippet que enviou, corrija ine-height para line-height. Nesta doc, o exemplo estático usa th-title-abbrv com ajuste só em design-system-docs.css (preview).
