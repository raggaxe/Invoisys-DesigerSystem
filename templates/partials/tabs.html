Visão geral

Há dois padrões principais: (1) a tab bar em fundo branco com .invoisys-tabbar (Figma / telas gerais) e (2) as abas do modal Detalhes em ul.nav.nav-tabs.modal-detalhes-tabs dentro de data-testid="modal-detalhes-container", com Bootstrap 3 (data-toggle="tab") no portal. No primeiro caso (.invoisys-tabbar), o item ativo usa texto escuro, negrito e linha inferior — o snippet copiável está na secção HTML · tab bar Figma. O exemplo interativo desta página usa só o padrão do modal Detalhes (nav-tabs), como nas restantes abas do portal.

Figma vs CSS: no arquivo do Figma costuma aparecer uma moldura roxa tracejada, largura fixa (1184px) e order / flex-grow do auto-layout — isso serve para o frame de desenho, não para produção. No portal usa-se só o markup e as classes abaixo; largura e margens vêm do layout da página.

Exemplo interativo · modal Detalhes (nav-tabs)

Na tela Detalhes do documento, as abas seguem o markup clássico do Bootstrap 3: ul.nav.nav-tabs com nav-primary e modal-detalhes-tabs, ícone Font Awesome + texto em negrito no link, e data-toggle="tab" apontando para div.tab-pane em .tab-content. Em produção, o antecessor data-testid="modal-detalhes-container" é obrigatório para os seletores em componentes.css; nesta doc a pré-visualização usa data-testid="ds-modal-detalhes-tabs-container" para não duplicar o mesmo valor no HTML (regras equivalentes em design-system-docs.css).

Storybook: a folha design-system-docs.css acrescenta disposição em linha das abas, realce da aba ativa e troca de painéis na pré-visualização; o design-system-docs.js simula o clique sem o bootstrap.js completo. Em MVC mantenha o init oficial do Bootstrap 3.

Os painéis colapsáveis por baixo das abas (Dados do Emitente, …) estão na página Colapso (.modal-detalhes-panel).

Outro padrão: tab bar Figma (invoisys-tabbar)

Isto não é o mesmo que as abas do modal Detalhes (as de cima, com ícones e nav-tabs). É um segundo padrão do design system: a barra clara do Figma para secções dentro de uma página ou módulo (ex.: alternar «Resumo», «Detalhes», «Histórico» numa área de trabalho), usando só div + button e as classes .invoisys-tabbar / .invoisys-tabbar__tab definidas em componentes.css.

O que fazer com este bloco: copia o HTML/Razor para a tua view quando o layout for tab bar Figma. Não há pré-visualização aqui porque o exemplo interativo desta página foi unificado com o modal; o snippet serve só como referência copiável deste outro padrão.

Razor / HTML 
<div class="invoisys-tabbar" role="tablist" aria-label="Secções da área de trabalho">
    <div class="invoisys-tabbar__track">
        <button type="button" class="invoisys-tabbar__tab invoisys-tabbar__tab--active" role="tab" aria-selected="true">Resumo</button>
        <button type="button" class="invoisys-tabbar__tab" role="tab" aria-selected="false">Detalhes</button>
        <button type="button" class="invoisys-tabbar__tab" role="tab" aria-selected="false">Histórico</button>
    </div>
</div>

Comportamento (tab bar): em MVC, troca invoisys-tabbar__tab--active e aria-selected no servidor ou via JavaScript ao mudar de separador; mantenha o foco e o painel associado alinhados às práticas ARIA de tabs se você ligar a tabpanel.
