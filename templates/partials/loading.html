Visão geral

O estado «carregando» com animação em listagens (por exemplo com DataTables) combina o GIF de marca (~/Images/loader.gif) com texto em .heading-md e .fw-bold, dentro de .loading-table definido em table.css (fundo com loading-pulse, conteúdo centrado). Em painéis que não são só tabela, o portal também pode usar .loading-panel — o mesmo ritmo visual.

CSS: inclua table.css na view (cascade em Introdução — ordem de cascade nas views). Placeholders sem GIF ficam em Skeletons (.skeleton-pulse, etc.) — esta página cobre o padrão loader + mensagem.

Acessibilidade: com texto visível, reforce o estado no contêiner (aria-busy="true", role="status"). O GIF é decorativo: alt="" e, se preferir, aria-hidden="true" na img.

HTML · Razor (overlay genérico)

O padrão mínimo é .loading-table + GIF + mensagem. Para mostrar ou ocultar o overlay, use classes de estado com prefixo do seu módulo (ex.: pedidos-loading, relatorios-loading) e controle-as no JavaScript da listagem — o mesmo critério vale para data-testid estável nos testes.

Dentro de .layout-datatable-page, section[data-table] e .panel-content.pagination2, o table.css do portal pode amarrar posicionamento absoluto do overlay a um conjunto específico de classes legadas. Ao criar uma listagem nova, replique a estrutura de markup da grade e alinhe com a equipe de front-end se precisar de regras equivalentes para o seu prefixo.

Razor / HTML 
<div class="loading-table seu-modulo-loading seu-modulo-loading--visivel"
     style="display: none;"
     role="status"
     aria-busy="true"
     data-testid="seu-modulo-loading">
    <img src="~/Images/loader.gif" width="75" alt="" />
    <div class="heading-md fw-bold">Carregando…</div>
</div>

Em runtime, o style="display: none;" ou uma classe do tipo seu-modulo-loading--oculto é controlado pelo JS da view — não copie um estado «sempre visível» para produção se o fluxo começa oculto.
