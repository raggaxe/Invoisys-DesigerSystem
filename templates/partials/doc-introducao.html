Introdução

Esta documentação acompanha o design system Invoisys: a referência viva para quem constrói telas em C# · ASP.NET MVC · Visual Studio, com folhas em Content/Css e scripts de apoio (por exemplo em Scripts/Tabelas).

Para que serve

A intenção é facilitar o dia a dia de quem desenha e de quem implementa: menos decisões ad hoc, menos retrabalho e uma experiência do usuário previsível em todo o portal. O design system concentra tokens, utilitários e componentes já alinhados com UX e produto — este site é o lugar onde se vê o comportamento, se copia o markup e se alinham discussões antes de tocar no código oficial do portal.

Para desenvolvimento, o ganho é direto: exemplos em HTML/Razor alinhados ao que o CSS do projeto já suporta, com a cascade documentada abaixo e snippets prontos para colar no Visual Studio. Para reutilização, as views deixam de reinventar botões, modais ou listagens — compõem-se a partir das mesmas classes e padrões, o que reduz dívida visual e acelera code review.

Atualização dinâmica dos componentes

Quando o token ou o componente muda nos arquivos oficiais de CSS do portal (main.css, tokens.css, componentes.css, table.css no repositório), todas as views que seguem o padrão beneficiam da mesma alteração — sem caçar estilos inline ou CSS duplicado por tela.

Este repositório de documentação / storybook não substitui o fluxo de alteração do design system no produto: serve para validar, documentar e testar antes (e depois) de integrar no MVC. A ordem de cascade nas views e as prioridades desta documentação estão nas seções abaixo. Missão, valores e propósito em torno de reutilização, consistência e manutenção estão em Manifesto de UX e cultura da equipe; comandos, links e arquivos para baixar estão no Quick start.

Componentes e prioridades

Páginas cobertas nesta doc: Cabeçalho, Colapso, Tabs, Timeline, Inputs, Loading, Skeletons, Cards, Alertas, Pills, Botões, Tooltips, Modais, Ícones, …

Prioridade em listagens. Em Inputs, trate Select2 e intervalo de datas com o mesmo nível de integração (jQuery, vendors, cascade).

Sem CSS novo na view. Use tokens + componentes (+ table quando houver listagens) e combine classes já existentes.
